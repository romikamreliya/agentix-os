# Design Trend Research (Phase 4)

Browse the web to ground the visual system in what's current and effective. Run
this **after** preferences (Phase 1), product analysis (Phase 2), and the UX flow
(Phase 3) so queries are targeted — research informs the design, it never
overrides the user's stated preferences or the product's goals.

## What to research
1. **Latest design trends** for this product type and audience (e.g. "SaaS
   dashboard design trends 2026", "fintech app UI trends", "modern landing page
   patterns").
2. **Innovative / emerging patterns** — new interaction models, layout systems,
   color and typography directions worth considering.
3. **Modern UI examples** — real shipping products, design galleries, and
   component libraries that fit the Phase 1 style. Analyze *what* makes each work
   (hierarchy, color use, spacing, motion), not just that it looks nice.
4. **Improvement ideas** — specific, adoptable changes that would strengthen the
   visual design and overall UX for this product.

## How to browse (delegate via the Skill tool)
- **`firecrawl-search`** — discover trend articles, galleries, and examples from
  a query; returns full page content, not just snippets.
- **`firecrawl-scrape`** — pull the content of a specific article or example page.
- **`firecrawl-website-design-clone`** — when the user points at a specific site,
  extract its design system (colors, type, spacing, components) into a DESIGN.md.
- **`firecrawl-deep-research`** — for a rigorous, cited survey when the user wants
  a thorough trend report.
- **`WebSearch` / `WebFetch`** — fallbacks if the firecrawl skills are unavailable.

Good sources to consider: design galleries (Dribbble, Behance, Awwwards, Mobbin,
Land-book, SaaS/Page collections), component libraries and their showcases,
reputable design blogs, and direct competitor products.

## Turning findings into recommendations
For each relevant finding, capture:
- **The pattern/idea** in one line, with a **source link**.
- **Why it fits** this product's goals, audience, or the user's stated taste.
- **How to apply it** — the concrete change to color, type, layout, component, or
  interaction it suggests for Phase 5.

## Guardrails
- Keep it targeted: **3–6 strong findings**, not an exhaustive dump.
- Respect Phase 1 preferences — surface a trend that conflicts as an *option*,
  don't silently adopt it.
- Prefer timeless usability over novelty; flag trends that hurt accessibility or
  performance (e.g. low-contrast fashions, heavy motion) rather than recommending
  them.
- Note the date/recency of sources; trends age.

## Output of Phase 4
A short **trend brief**: 3–6 findings, each with a source, a fit rationale, and a
concrete application — plus which ones you recommend adopting. This feeds directly
into the visual-system choices in Phase 5.
